# 基于Python与PySide6的音乐播放器

## 项目概述
使用 **PySide6**（Qt for Python）开发一款跨平台（Windows / macOS / Linux）本地音乐播放器，支持：
- 本地音频文件播放（MP3, WAV, WMA, AAC, FLAC, AU 等全格式）
- 播放列表管理及多种播放模式（顺序/随机/单曲循环/列表循环）
- 调用外部 API 搜索在线音乐，可添加到播放列表或下载到本地
- 桌面歌词（无边框置顶窗口，LRC 解析与同步）

**核心设计原则**：
- 不依赖系统媒体框架，统一使用 **FFmpeg** 解码 + **PySide6 QAudioOutput** 播放，确保全平台行为一致。
- UI 层与业务逻辑严格分离，通过信号/槽通信。
- 所有第三方库通过 Python 包或系统包管理器提供，避免硬编码路径。

## 架构与关键模块设计

### 整体通信
所有模块继承 `QObject`，使用 PySide6 信号/槽机制解耦。  
主窗口 (`MainWindow`) 负责连接核心模块的信号到 UI 更新函数，不直接调用核心模块的方法。

### 音频引擎
- 使用 PyAV 打开音频文件并获取流。
- 解码音频帧，重采样为 `QAudioFormat` 要求的格式（推荐 44100 Hz、立体声、16 位有符号整数）。
- 将解码后的原始 PCM 字节流写入 `QIODevice`，提供给 `QAudioOutput` 播放。
- **解码必须在独立的 QThread 中执行**，以防阻塞 UI。
- 暴露方法：`play()`、`pause()`、`stop()`、`seek(position_ms)`、`set_volume(float)`。
- 定期发送信号 `position_changed`（当前播放位置，毫秒）和 `state_changed`（播放/暂停/停止状态）。

### 播放列表管理
- 使用 SQLite 数据库存储歌曲、播放列表以及列表与歌曲的关联关系。
- 关键数据表：`songs`（文件路径、标题、艺术家、专辑、时长、格式、哈希）、`playlists`（名称、创建时间）、`playlist_songs`（列表ID、歌曲ID、排序索引）。
- 管理当前播放列表、当前播放索引。
- 提供方法：添加歌曲、移除歌曲、下一首、上一首、切换播放模式、保存/加载播放列表。
- **播放模式**：顺序播放、列表循环、单曲循环、随机播放。随机模式使用洗牌算法生成随机播放序列。
- 当引擎播放完一首歌时，根据当前模式自动计算下一首并通知引擎加载。

### 本地音乐入库
- 遍历指定目录（包含子目录），利用 `mutagen` 提取元数据。
- 计算文件哈希以避免重复添加。
- 将歌曲信息批量插入数据库，并通过信号报告扫描进度。

### 在线服务
- 定义抽象基类 `BaseSearchProvider`，声明搜索方法和获取下载链接的方法。
- 具体实现针对外部音乐 API，使用 `QNetworkAccessManager` 发送异步 HTTP 请求，解析 JSON 响应。
- 搜索结果返回为统一的 `Song` 对象列表。
- 下载歌曲时采用异步流式写入本地文件，完成后更新数据库记录并关联到播放列表。

### 桌面歌词
- 解析标准 LRC 文件，构建按时间排序的行列表（每项包含时间毫秒和歌词文本）。
- 接收来自引擎的 `position_changed` 信号，查找当前应高亮的歌词行索引。
- 歌词窗口使用无边框、置顶、工具窗口样式，支持鼠标拖拽移动和右键菜单（锁定、字体调整、关闭）。
- 绘制时采用双缓冲，当前行高亮，背景半透明，上下行渐变效果。

## 编码规范

- 使用 Python 3.10+ 的类型提示，例如为方法参数和返回值标注类型。
- 类名用 PascalCase，函数名、变量名用 snake_case。
- 内部方法或私有属性以单下划线开头。
- 每个类的信号应在 `__init__` 方法的最顶部集中定义。
- 所有源文件使用 UTF-8 编码。
- 禁止使用 `from module import *`。
- 注释用英文说明“为什么这样做”，而不是简单复述代码。
- 数据库操作一律使用参数化查询，防止 SQL 注入。
- 所有可能抛出异常的操作（IO、网络、解码）必须用 try-except 捕获，并通过信号将错误信息传递到 UI 层，不得静默失败。
- 要播放音频测试前务必用中文告知用户。
